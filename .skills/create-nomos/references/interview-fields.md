# Interview Fields

Interview for **pack-level methodology**, not for one project's current facts.

## Required questions

1. What kind of projects is this Nomos meant for?
2. Which constitution principles are universal for projects using this pack?
3. Which docs harness layers are required?
4. Which lifecycle modules are mandatory?
5. Which actions require human approval?
6. What should bootstrap always ask and fill?
7. What should closeout always harvest?
8. What should doctor always verify?

## Avoid these traps

- Do not ask for current repo paths as if they were pack metadata.
- Do not bake one team's private planning documents into the pack.
- Do not mix install instructions with project-state content.
