# Agora Public Docs

`Doc/` is the public, shareable documentation surface that lives in the main repository.

It now mirrors the private `docs/` information architecture so external contributors can follow the same working shape without seeing internal task history or maintainer-only records.

## What lives here

- public onboarding, setup, and architecture docs
- contributor workflow rules and public reference mirrors
- public planning skeletons, templates, and safe examples
- example walkthroughs and delivery records that contributors can copy

## What does not live here

- internal planning history
- private reviews and audit notes
- maintainer-only implementation logs
- sensitive operational detail that should stay in the separate private `docs/` repo

## Scope Rules

- Put tutorials, architecture overviews, contributor rules, and reusable templates here.
- Use the mirrored folders to show contributors where plans, findings, progress logs, and walkthroughs belong.
- Keep the private `docs/` repo as the maintainer source of truth for active internal task records.
