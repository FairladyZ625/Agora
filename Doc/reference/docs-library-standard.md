# Docs Library Standard

## Purpose

Explain where public and private documentation should live.

## Public Docs

Use `Doc/` for:

- onboarding guides
- tutorials
- quick starts
- contributor references
- architecture overviews that are safe to share publicly

## Private Docs

The separate private `docs/` repository is used by maintainers for:

- planning logs
- internal walkthroughs
- governance details
- implementation history
- deeper architecture records that are not part of the public bundle

## Rule of Thumb

If an external contributor needs it to understand the project or submit a PR, put a public version in `Doc/`.
