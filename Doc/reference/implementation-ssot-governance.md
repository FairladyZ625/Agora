# Implementation SSoT Governance

## What This Means Publicly

Agora maintainers use a private implementation control-tower document and linked planning records to track active work.

External contributors do not need direct access to that internal SSoT to contribute successfully.

## What Contributors Should Do Instead

- Keep PR descriptions precise.
- Link issues, design notes, or public docs that explain the change.
- State what was verified.
- Avoid introducing parallel roadmap or status files in the repository root.

## Why This Exists

The internal SSoT prevents planning drift for maintainers. The public repository should stay focused on code, public docs, and contributor clarity.
