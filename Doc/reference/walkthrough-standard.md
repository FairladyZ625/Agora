# Walkthrough Standard

## Goal

Summaries should explain what changed, how it was verified, and what remains out of scope.

## Good Delivery Notes Include

- the user-visible or maintainer-visible outcome
- the key files or surfaces touched
- the verification commands that were run
- remaining risks or deferred work

## Public Contribution Equivalent

For external contributors, the pull request description is the main public walkthrough. Write it so maintainers can review the change without guessing your intent or validation steps.
